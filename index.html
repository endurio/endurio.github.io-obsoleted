Economic-driven Sharding Protocol

Yggdrasil sharding protocol dynamically split the workload to more shards as the chain is reaching saturation. Input Locking Service provides instant confirmation with 100% warranty no double-spend for all transactions. This allows Endurio to have an wonderful user experience while keeping the blocktime long enough to preserve the security and consistency of the chain.

Technical

1 You have your own sharding protocol? How does it compare to Ethereum Sharding?

Our sharding protocol is economic-driven. It sidesteps many challenges other pure-technical sharding protocol (like Ethereum) has to faces, including:

  • Scalability of super-full and top-level nodes (where all state from all shards is verified).
  • High frequency of cross-shard communication and transactions.

Yggdrasil sharding protocol has no super-full node, each node always works on only one shard at a time, and most of the transactions are occurred locally in one shard. These features make itâ€™s extremely efficient scaling solution, just like the sharding architect of traditional database.

2 How is instant confirmation done?

By locking every inputs of the transaction. Each input can only be locked by a single SSN node, so if thereâ€™s a double-spend, there must be conflicted signatures by that node, which can be punished.
